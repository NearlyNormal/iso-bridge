    About ISO Bridge

    ISO Bridge is a country-aware payments format translation workbench. It converts between SWIFT MT legacy messages and ISO 20022 XML, showing exactly what happens to every field in the process.

    Why this matters now. The SWIFT deadline for ISO 20022 adoption on cross-border payments was November 2025. In practice, the migration is not uniform. Large banks have implicit extensions. Mid-size institutions are running translation middleware that converts ISO 20022 back to MT for internal consumption, buying time while deeper integration work continues. The result is a coexistence period where both formats live in the same transaction chains, and data is lost at every translation boundary.

    A payment that originates as a pacs.008 may be translated to MT103 at a correspondent bank's gateway, processed internally as MT, and translated back to ISO 20022 at the receiving end. Structured remittance collapses into free text. Addresses lose their structure. Charge bearer codes approximate rather than map cleanly.

    This tool is honest about that data loss. Every translation shows:

    • Which fields map cleanly (green)
    • Which fields are transformed or approximated (amber)
    • Which data is lost or can't be represented (red)
    • Which ISO fields are auto-generated with no MT source (blue)

    Country-aware. Different countries are at different stages. The US has six active payment rails, some native ISO 20022 (RTP, FedNow), some mid-migration (Fedwire, CHIPS). Canada's Lynx has been native ISO 20022 since 2021. Brazil's PIX is built entirely on ISO 20022, with over 140 million users. The tool knows which formats are relevant in which market, so it only shows what you will actually encounter.

    Config-driven architecture. All country data, translation paths, and field-by-field mappings live in plain JavaScript configuration files. Adding a new country or a new format pair means editing a config file, never touching the engine. The configs are human-readable documentation in their own right.

    v1.0 coverage: MT103 ↔ pacs.008, MT202 ↔ pacs.009, MT940 ↔ camt.053 translation across US, Canada, and Brazil.

    Part of the payments toolkit:

    • SWIFT MT Parser decodes MT messages into structured JSON
    • ISO 20022 Validator validates XML against schemas and scheme profiles
    • ISO Bridge (this tool) translates between formats
